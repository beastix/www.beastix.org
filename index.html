The Beastix project

About Beastix

Beastix is a project to create a unix-like operating system out of 100% free software that looks and feels like a BSD system while using the Linux kernel and various GNU components. The project is currently under development with a 0.1 release coming soon, when complete it will combine the BSD feel with the software and hardware support of the Linux kernel.

100% Free software

Beastix is intended to be a completely free software operating system, that means respecting the 4 freedoms. You as the end user have complete control over your computing if you use Beastix.

Sub projects

Beastix is made up of the following sub projects:

  • Beastix - The Beastix base system itself contains everything you need to build a working OS
  • Beastix ports - A ports tree for Beastix originated from NetBSD pkgsrc
  • BSD tools - An ongoing effort to replace busybox with a full BSD toolset based on code from FreeBSD and NetBSD

In addition Beastix integrates the following 3rd-party components (with local modifications) as part of the base system:
  • Linux kernel version 3.19 with linux-libre patches and security fixes
  • GNU binutils version 2.22 patched with musl support
  • GNU gcc version 4.7.2 patched with musl support
  • Busybox version 1.23.2
  • musl libc version 1.0.5 with local extensions

Please note that third-party components have been modified in various ways in order to work well as part of the whole system, it is not recommended to use them outside of Beastix.

Where to get it

You will find the source code on github at https://github.com/beastix. To build from source you will need an existing GNU/Linux distro (or Beastix itself). See the README included in the source tree for instructions on building.

Alternatively, you can use BitTorrent to download an ISO image of the latest build

Helping out

Beastix needs help! If you would like to help out, send an email to joinus@beastix.org. Alternatively, just submit pull requests and bug reports on github. We need help in the following areas:

  • Testing - just download, build and play with Beastix and see below for instructions on reporting bugs
  • Developers - we need people to work on the codebase in general, and on porting BSD tools in particular
  • Security - Beastix needs some quality security auditing
  • Website - It should be obvious what's needed here

You can also find the core developers on IRC in #beastix on irc.freenode.net

Bug reports

Bug reports should be submitted as issues on the relevant github project, pull requests containing patches are most useful, but aside from that please ensure bug reports contain as much information as possible about your machine and what you were trying to do. Include steps on how to reproduce and full console logs where possible.

If you find nonfree software in Beastix, it is sufficient to simply link to it in a github issue and identify in what way it is nonfree.
